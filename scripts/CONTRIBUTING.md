## 🚀 ¿Cómo empezar?

1. Hacé un fork del repo
2. Cloná tu fork
3. Creá una nueva rama para tu feature:
   ```bash
   git checkout -b mi-feature
git commit -m "Agrego nueva funcionalidad X"
git push origin mi-feature
Creá un Pull Request en GitHub 🎉

📦 Buenas Prácticas
Escribí commits claros y descriptivos

Usá nombres de ramas con prefijo (fix/, feature/, hotfix/)

Evitá subir archivos innecesarios (ej: node_modules/, archivos de sistema)

Si modificás un script o proceso, actualizá el README del mismo

🛡 Revisión y Aprobación
Todos los PR serán revisados por el equipo.

Podés dejar comentarios explicando el propósito del cambio.

Si hay conflictos, te pediremos actualizarlos antes del merge.

¡Gracias por colaborar! Cualquier duda, no dudes en contactar al equipo técnico 🙌
